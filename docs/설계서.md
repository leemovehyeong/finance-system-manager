## 2. UX/UI 디자인 가이드 — Apple HIG 감성

> B2B 시스템의 투박함을 벗어던지고, 하이엔드 라이프스타일 앱 감성으로 만든다.

### 2.1 Typography & Space

- 폰트: SF Pro (system-ui, -apple-system 폴백)
- 텍스트 위계를 크기와 굵기(Weight)로만 구분. 색상으로 구분하지 않음.
- 여백(White Space) 과감하게. 화면이 숨 쉴 수 있도록 Editorial design vibe.
- 제목: text-2xl font-semibold tracking-tight
- 부제목: text-lg font-medium
- 본문: text-base
- 캡션: text-sm text-[#8E8E93]

### 2.2 Color Palette & Material

- 배경: #FFFFFF / #F2F2F7 (iOS Off-white)
- 메인 텍스트: #1C1C1E (순수 블랙 #000000 사용 금지)
- 서브 텍스트: #8E8E93
- Accent: #007AFF (iOS Blue) — 주요 CTA
- 상태색: 대기 #FF9500 / 처리중 #007AFF / 완료 #34C759 / 긴급 #FF3B30 / 취소 #8E8E93
- 카드 및 모달에 iOS 블러 효과(Glassmorphism)와 넓고 부드러운 그림자

### 2.3 Component Style

- Card: rounded-2xl, shadow(0 2px 15px rgba(0,0,0,0.06)), p-5
- Button Primary: bg-[#007AFF], rounded-xl, h-[52px], font-semibold, press시 scale(0.97)
- Button Secondary: bg-[#F2F2F7], text-[#007AFF]
- Input: bg-[#F2F2F7], rounded-xl, h-[48px], border-none (focus: ring-2 ring-blue-500)
- Badge: rounded-full, 상태색 10% opacity bg + 원색 text, px-3 py-1
- NavBar: backdrop-blur-xl, bg-white/72, border-t-[0.5px] border-black/8
- Modal: Bottom Sheet, blur backdrop, top-rounded-[20px], drag handle
- 복잡한 표(Table) 대신 카드형 리스트 사용
- 데이터 시각화: 미니멀 프로그레스 바, 도넛 차트, 부드러운 진입 애니메이션
- 모든 모서리 부드러운 곡선. iOS 특유의 둥글둥글한 느낌.

### 2.4 Interaction

- 모든 터치 요소: press 피드백 (scale 0.97, opacity 0.7)
- 페이지 전환: slide from right
- 로딩: skeleton screen (spinner 사용 금지)
- Pull to refresh
- Swipe-to-action (삭제, 완료 등)

### 3.1 카카오맵 — 앱 내 지도 (핵심 기능)

카카오 개발자 등록 → JS 앱 키 → NEXT_PUBLIC_KAKAO_MAP_KEY 환경변수.

지도는 외부 앱으로 연결하지 않는다. 우리 앱 안에서 직접 표시한다.

기능:
1. 앱 내 지도에 거래처 위치 마커 표시
2. 오늘의 A/S, 설치 건을 실시간 마커로 표시 (상태별 색상)
3. "동선 최적화" 버튼 → 수락한 업무들의 주소 기반 최적 경로 자동 계산
4. 마커 클릭 → 티켓 상세로 이동
5. "카카오내비로 출발" → 카카오맵 네비 앱 연결 (출발만 외부)

마커 색상:
- 🔴 긴급 대기
- 🟡 일반 대기
- 🔵 처리 중
- 🟢 설치 예정

## 5. 데이터베이스 스키마

### 5.1 employees

```sql
CREATE TABLE employees (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  auth_id UUID REFERENCES auth.users(id), -- Supabase Auth 연결
  name TEXT NOT NULL,
  email TEXT UNIQUE NOT NULL,
  phone TEXT,
  role TEXT NOT NULL CHECK (role IN ('admin', 'office', 'field')),
  profile_image TEXT,
  is_active BOOLEAN DEFAULT true,
  created_at TIMESTAMPTZ DEFAULT now()
);
```

### 5.2 stores (거래처)

```sql
CREATE TABLE stores (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  business_number TEXT,
  owner_name TEXT,
  phone TEXT,
  address TEXT,
  latitude DOUBLE PRECISION,
  longitude DOUBLE PRECISION,
  region TEXT CHECK (region IN ('여수', '순천', '광양', '기타')),
  has_card_terminal BOOLEAN DEFAULT false,
  has_pos BOOLEAN DEFAULT false,
  has_kiosk BOOLEAN DEFAULT false,
  has_table_order BOOLEAN DEFAULT false,
  terminal_count INTEGER DEFAULT 0,
  pos_count INTEGER DEFAULT 0,
  franchise TEXT,
  van_type TEXT,
  contract_date DATE,
  memo TEXT,
  last_visit_at TIMESTAMPTZ,
  visit_count INTEGER DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT now()
);

CREATE EXTENSION IF NOT EXISTS pg_trgm;
CREATE INDEX idx_stores_name ON stores USING gin(name gin_trgm_ops);
CREATE INDEX idx_stores_region ON stores(region);
```

### 5.3 tickets (업무 티켓)

```sql
CREATE TABLE tickets (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  ticket_number SERIAL,
  store_id UUID REFERENCES stores(id),
  store_name TEXT NOT NULL,
  store_phone TEXT,
  store_address TEXT,
  type TEXT NOT NULL CHECK (type IN ('as', 'install', 'paper', 'inspect', 'sales', 'other')),
  priority TEXT DEFAULT 'normal' CHECK (priority IN ('urgent', 'normal', 'low')),
  title TEXT NOT NULL,
  description TEXT,
  equipment_type TEXT CHECK (equipment_type IN ('card_terminal', 'pos', 'kiosk', 'table_order', 'printer', 'other')),
  equipment_detail TEXT,
  paper_type TEXT CHECK (paper_type IN ('pos', 'thermal', 'portable')),
  paper_quantity INTEGER,
  status TEXT DEFAULT 'pending' CHECK (status IN ('pending', 'accepted', 'in_progress', 'completed', 'cancelled')),
  source TEXT DEFAULT 'internal' CHECK (source IN ('internal', 'external')),
  created_by UUID REFERENCES employees(id),
  assigned_to UUID REFERENCES employees(id),
  completion_memo TEXT,
  completed_at TIMESTAMPTZ,
  images TEXT[],
  created_at TIMESTAMPTZ DEFAULT now(),
  accepted_at TIMESTAMPTZ,
  updated_at TIMESTAMPTZ DEFAULT now()
);

CREATE INDEX idx_tickets_status ON tickets(status);
CREATE INDEX idx_tickets_type ON tickets(type);
CREATE INDEX idx_tickets_source ON tickets(source);
CREATE INDEX idx_tickets_assigned_to ON tickets(assigned_to);
CREATE INDEX idx_tickets_created_at ON tickets(created_at DESC);
```

## 7. 업무 흐름

### 7.1 A/S/일반 티켓

```
전화 또는 외부접수 → 사무직 접수 등록 (또는 자동 등록) →
알림 → 전체 현장직 피드 → 현장직 "수락" →
상태: accepted → 방문/처리 → "완료" + 메모/사진 →
상태: completed → 거래처 이력 갱신
```

### 7.2 영업/설치

```
영업 프로젝트 생성 (prospecting) →
영업 체크리스트 진행 →
계약 완료 (contracted) → 서류 업로드 →
설치 일정 확정 (install_scheduled) → 현장직 배정/알림 →
설치 진행 (installing) → 현장 체크리스트 →
설치 완료 (completed) → 사진 업로드 → 거래처 자동 등록
```

### 7.3 영업 체크리스트

사업자등록증 수취, 계약서 작성, 매장 실측, 메뉴 확인, 프로그램 선정, VAN사 선정, 카운터/인터넷 확인, 오픈예정일 확인

### 7.4 현장 설치 체크리스트

장비 출고 확인, 카운터 설치, 인터넷 연결, POS 프로그램 설치, 카드단말기 연동, 테스트결제, 프로그램 사용법 교육, 오늘얼마 어플 설치, 설치완료 사진 업로드, 재부팅/OS 구동 확인

## 8. 외부 접수

- /request: 범용 QR (명함, 스티커)
